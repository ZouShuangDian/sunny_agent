## 二、完整消息流程（10个阶段）

### 阶段1️⃣：飞书消息接收

```
┌────────────────────────────────────────────────────────────────┐
│                    Feishu WebSocket连接                         │
└────────────────────────────────────────────────────────────────┘

Feishu Plugin (extensions/feishu/)
│
├─► 创建 Lark.WSClient
│     ├─ appId: xxx
│     ├─ appSecret: xxx
│     └─ domain: feishu.cn / larksuite.com
│
├─► 注册事件处理器
│     └─ im.message.receive_v1
│
└─► 启动 WebSocket 连接
      └─ await wsClient.start()

        │
        ▼
飞书服务器推送消息事件
        │
        ▼
触发 processFeishuMessage()
```

### 阶段2️⃣：消息解析

```
┌────────────────────────────────────────────────────────────────┐
│                    飞书消息格式解析                             │
└────────────────────────────────────────────────────────────────┘

飞书消息结构 (src/feishu/message.ts)
│
├─► message.chat_id        ──► 聊天ID
├─► message.chat_type      ──► group / p2p
├─► message.message_type   ──► text / image / file
├─► message.content        ──► JSON格式内容
├─► message.mentions       ──► @提及列表
├─► message.create_time    ──► 时间戳
└─► message.message_id     ──► 消息ID

        │
        ▼
提取关键信息
        │
        ▼
┌────────────────────────────────────────────────┐
│ chatId     : oc_xxx / ou_xxx                  │
│ isGroup    : true / false                     │
│ senderId   : ou_xxx (open_id)                 │
│ msgType    : text / image / file              │
│ text       : "消息内容"                        │
└────────────────────────────────────────────────┘
```

### 阶段3️⃣：准入控制（多层过滤）

```
┌────────────────────────────────────────────────────────────────┐
│                    消息准入控制流程                             │
└────────────────────────────────────────────────────────────────┘

消息到达
   │
   ├─► [检查1] 消息类型支持?
   │       ├─► text ✓
   │       ├─► image ✓
   │       ├─► file ✓
   │       └─► other ✗ ───────────► 丢弃
   │
   ├─► [检查2] 是群组消息?
   │       │
   │       ├─► 群组启用? ──► 否 ──► 丢弃
   │       │
   │       ├─► 群组白名单检查
   │       │       ├─► 在白名单 ──► 继续
   │       │       └─► 不在名单 ──► 丢弃
   │       │
   │       └─► @提及检查
   │               ├─► 已@Bot ──► 继续
   │               └─► 未@且requireMention ──► 丢弃
   │
   └─► [检查3] 是私聊消息?
           │
           ├─► dmPolicy = disabled ──► 丢弃
           │
           ├─► dmPolicy = allowlist
           │       ├─► 在列表 ──► 继续
           │       └─► 不在列表 ──► 丢弃
           │
           └─► dmPolicy = pairing
                   ├─► 已授权 ──► 继续
                   └─► 未授权 ──► 发送配对码 ──► 结束
```

### 阶段4️⃣：配对机制（未授权用户）

```
┌────────────────────────────────────────────────────────────────┐
│                    设备配对流程                                 │
└────────────────────────────────────────────────────────────────┘

未授权用户发送消息
        │
        ▼
生成配对请求
├─► 提取 open_id: ou_xxx
├─► 提取 union_id: on_xxx
└─► 生成配对码: ABC123

        │
        ▼
发送配对消息给用户
┌────────────────────────────────────────────────────────┐
│ OpenClaw access not configured.                        │
│                                                        │
│ Your Feishu Open ID: ou_xxx                           │
│                                                        │
│ Pairing code: ABC123                                   │
│                                                        │
│ Ask the OpenClaw admin to approve with:               │
│ openclaw pairing approve feishu ABC123                │
└────────────────────────────────────────────────────────┘

        │
        ▼
等待管理员审批
├─► 执行: openclaw pairing approve feishu ABC123
├─► 添加到 allowFrom 列表
└─► 用户获得访问权限

        │
        ▼
用户再次发送消息 ──► 通过检查 ──► 继续处理
```

### 阶段5️⃣：构建消息上下文

```
┌────────────────────────────────────────────────────────────────┐
│                    MsgContext 构建                             │
└────────────────────────────────────────────────────────────────┘

processFeishuMessage() 构建上下文对象
│
├─► Body:           "消息内容"
├─► RawBody:        "原始消息"
├─► From:            "ou_xxx" (sender_open_id)
├─► To:              "oc_xxx" (chat_id)
├─► SenderId:        "ou_xxx"
├─► SenderName:      "用户名"
├─► ChatType:        "dm" / "group"
├─► Provider:        "feishu"
├─► Surface:         "feishu"
├─► Timestamp:       1704067200000
├─► MessageSid:      "om_xxx"
├─► AccountId:       "default"
├─► OriginatingChannel: "feishu"
├─► OriginatingTo:   "oc_xxx"
├─► MediaPath:       "/path/to/media" (如有)
└─► MediaType:       "image/jpeg" (如有)

        │
        ▼
dispatchReplyWithBufferedBlockDispatcher(ctx, cfg, options)
```

### 阶段6️⃣：Session 初始化

```
┌────────────────────────────────────────────────────────────────┐
│                    Session 管理流程                             │
└────────────────────────────────────────────────────────────────┘

initSessionState()
│
├─► 1. 解析 Session Scope
│       ├─► per-sender: 每个用户独立会话
│       └─► global: 所有用户共享会话
│
├─► 2. 确定 Agent ID
│       ├─► 从 binding 路由查找
│       ├─► 或使用 default Agent
│       └─► 或使用第一个 Agent
│
├─► 3. 生成 Session Key
│       │
│       ├─► 私聊: agent:main:feishu:direct:ou_xxx
│       └─► 群组: agent:main:feishu:group:oc_xxx
│
├─► 4. 加载 Session Store
│       └─► ~/.openclaw/agents/main/sessions/sessions.json
│
├─► 5. 检查 Freshness
│       ├─► Idle 超时? ──► 重置 Session
│       ├─► Daily 重置? ──► 重置 Session
│       └─► Fresh ──► 复用 Session
│
└─► 6. 创建/复用 Session Entry
        ├─► sessionId: uuid
        ├─► updatedAt: timestamp
        └─► chatType: direct/group

Session 文件路径:
~/.openclaw/agents/main/sessions/{sessionId}.jsonl
```

### 阶段7️⃣：调用 Agent

```
┌────────────────────────────────────────────────────────────────┐
│                    Agent 调用流程                               │
└────────────────────────────────────────────────────────────────┘

runEmbeddedPiAgent()
│
├─► 1. 解析配置
│       ├─► 工作区: ~/.openclaw/workspace/
│       ├─► Agent目录: ~/.openclaw/agents/main/
│       └─► 模型: anthropic/claude-opus-4-5
│
├─► 2. 加载历史消息
│       └─► 从 Session .jsonl 文件读取
│
├─► 3. 构建工具集
│       ├─► read / write / edit
│       ├─► exec / bash
│       ├─► browser / canvas
│       ├─► message / cron
│       └─► ... (根据策略过滤)
│
└─► 4. 调用 Pi Agent Runtime
        └─► Pi RPC 调用

        │
        ▼
Agent 开始思考...
```

### 阶段8️⃣：事件流处理

```
┌────────────────────────────────────────────────────────────────┐
│                    事件订阅与广播                               │
└────────────────────────────────────────────────────────────────┘

Pi Agent Runtime 产生事件流
│
├─► session.subscribe(eventHandler)
│
└─► 事件类型:
    │
    ├─► message_start     ──► 开始生成回复
    ├─► message_update    ──► 增量文本 (流式)
    ├─► message_end       ──► 回复完成
    │
    ├─► tool_execution_start   ──► 开始调用工具
    ├─► tool_execution_update  ──► 工具输出
    ├─► tool_execution_end     ──► 工具完成
    │
    ├─► agent_start       ──► Agent开始运行
    ├─► agent_end         ──► Agent运行结束
    │
    ├─► auto_compaction_start  ──► 开始压缩历史
    └─► auto_compaction_end    ──► 压缩完成

        │
        ▼
Gateway 广播事件
├─► emitChatDelta()      ──► 发送增量文本
├─► emitToolExecution()  ──► 发送工具事件
└─► emitLifecycleEvent() ──► 发送生命周期事件

        │
        ▼
WebSocket 广播到所有订阅客户端
```

### 阶段9️⃣：流式卡片更新（飞书特色）

┌────────────────────────────────────────────────────────────────┐

│       └─► 显示"思考中..."

└─► 实时更新卡片内容
        │
        ▼
Agent生成文本流
        │
        ▼
逐步更新卡片
┌──────────────────────────────────────┐
│ 🤖 OpenClaw                          │
│                                      │
│ 正在思考...                          │
│                                      │
│ 这是回复的第一部分内容...            │
│ 这是回复的第二部分内容...            │
│ 这是回复的第三部分内容...            │
└──────────────────────────────────────┘
        │

## 三、Gateway 核心作用总结

### 3.1 连接层（Gateway 职责）

```
┌────────────────────────────────────────────────────────────────┐
│                    Gateway 连接管理                             │
└────────────────────────────────────────────────────────────────┘

┌────────────────┐    ┌────────────────┐    ┌────────────────┐
│  WebSocket     │    │   HTTP Server  │    │  Canvas Host   │
│   Server       │    │   Port 18789   │    │   Server       │
│                │    │                │    │                │
│ • 实时双向通信  │    │ • REST API     │    │ • Web界面      │
│ • 事件广播     │    │ • Webhook      │    │ • 管理控制台   │
│ • 多客户端支持 │    │ • OpenAI兼容   │    │                │
└───────┬────────┘    └───────┬────────┘    └────────────────┘
        │                     │
        └──────────┬──────────┘
                   │
              ┌────▼────┐
              │ Gateway │
              │  Core   │
              └────┬────┘
                   │
        ┌──────────┼──────────┐
        │          │          │
   ┌────▼───┐ ┌────▼───┐ ┌────▼───┐
   │  Auth  │ │ Router │ │Events  │
   │ 认证   │ │ 路由   │ │ 事件   │
   └────────┘ └────────┘ └────────┘
```

### 3.2 核心功能清单

| 功能模块 | 具体实现 | 作用 |
|---------|---------|------|
| **WebSocket服务** | `src/gateway/server.impl.ts` | 处理实时双向通信 |
| **HTTP服务** | `src/gateway/server-http.ts` | OpenAI API、Webhook、静态文件 |
| **消息路由** | `src/gateway/server-methods.ts` | 请求分发与权限检查 |
| **事件广播** | `src/gateway/server-broadcast.ts` | 向所有客户端推送事件 |
| **聊天状态** | `src/gateway/server-chat.ts` | 管理聊天事件与Agent回调 |
| **认证授权** | `src/gateway/auth.ts` | Token验证、设备签名、权限检查 |
| **客户端管理** | `src/gateway/server/ws-connection.ts` | WebSocket连接生命周期 |
| **节点注册** | `src/gateway/node-registry.ts` | 远程设备管理 |

## 六、总结

### 6.1 Gateway 在飞书场景中的核心作用

| 阶段 | Gateway 职责 |
|------|-------------|
| **接收** | 通过 Feishu Plugin 建立 WebSocket 连接，监听飞书事件 |
| **解析** | 解析飞书消息格式，提取关键信息（sender/chat/content） |
| **准入** | 执行多层准入控制（消息类型、白名单、配对机制） |
| **路由** | 构建 MsgContext，分发到正确的 Agent |
| **Session** | 管理 Session 生命周期（创建/复用/重置） |
| **调用** | 调用 Pi Agent Runtime，传递消息和上下文 |
| **事件** | 订阅 Agent 事件流，广播给所有客户端 |
| **回复** | 通过 Feishu Plugin 发送回复到飞书服务器 |

### 6.2 设计亮点

- ✅ **插件化架构** - 飞书作为插件接入，与其他通道统一接口
- ✅ **准入控制** - 多层过滤（消息类型/白名单/配对机制）
- ✅ **Session 管理** - 自动创建/复用/重置，支持多租户
- ✅ **事件驱动** - 异步架构，流式回复，实时性强
- ✅ **流式卡片** - 飞书特色功能，实时显示思考过程
- ✅ **配对机制** - 安全的设备授权流程
