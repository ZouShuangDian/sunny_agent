## 数据流图

### 1. 任务调度流程（arq）

```
arq Scheduler ──► 每秒触发（cron second='*/1')
    │
    ▼
查询 PG: next_run_at <= NOW() AND is_active = TRUE
    │
    ▼
对每个到期任务:
    │
    ├─ INSERT task_executions (status='queued')
    │
    ├─ UPDATE scheduled_tasks
    │   SET next_run_at = calc_next_run(cron)
    │
    ├─ COMMIT
    │
    └─ enqueue_job('execute_chat_task',
                   execution_id=...,
                   user_id=...,
                   parameters=...)
        │
        └─ arq 自动处理队列和延迟
```

### 2. 任务执行流程（arq Worker）

```
arq Worker ──► 启动
    │
    ├─ on_startup: 初始化数据库连接池
    │
    ├─ 自动消费队列（BRPOP）
    │
    └─ 获得任务: execute_chat_task
        │
        ├─ 1. 幂等性检查（Redis）
        │   ├─ completed? 返回缓存结果
        │   └─ processing? raise Retry(defer=60)
        │
        ├─ 2. 更新 PG: status='running', worker_id
        │
        ├─ 3. 调用 Chat API
        │
        ├─ 4. 成功?
        │   ├─ 是: UPDATE PG status='completed', result
        │   │       记录 metrics
        │   │       return result
        │   │
        │   └─ 否: raise Retry(defer=backoff)
        │           arq 自动处理重试
        │           超过 max_tries 标记为 FAILED
        │
        └─ on_shutdown: 清理资源
```

## 关键优势

```
┌─────────────────────────────────────────────────────────────────┐
│                   使用 arq 的核心优势                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ✅ 无需自建时间轮                                               │
│     arq 内置 Cron 调度，支持秒级精度                             │
│                                                                 │
│  ✅ 无需分布式锁                                                 │
│     arq 自动处理队列同步，无需 Leader 选举                        │
│                                                                 │
│  ✅ 无需自建延迟队列                                             │
│     arq 内置延迟任务（_defer_by 参数）                            │
│                                                                 │
│  ✅ 无需手动管理重试                                             │
│     raise Retry()，arq 自动处理指数退避                           │
│                                                                 │
│  ✅ 无需自建心跳和健康检查                                        │
│     arq 内置 health_check_interval 和优雅关闭                     │
│                                                                 │
│  ✅ 代码量减少 80%                                               │
│     从 ~3000 行降至 ~500 行                                      │
│                                                                 │
│  ✅ 可靠性提升                                                   │
│     经过生产验证的框架 vs 自建代码                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
