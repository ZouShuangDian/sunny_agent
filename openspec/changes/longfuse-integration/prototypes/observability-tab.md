管理面板 → Trace 观测

- **两步自动化流程**：
  - 第一步：选择服务来源
    - **内置服务**：启动系统内置的 Langfuse v3（含完整依赖栈）

  - 第二步：自动初始化（服务连接成功后自动触发，API Key 自动写入 .env）

### 3. 功能说明

| 元素 | 说明 |
|------|------|
| 起始日期 | 日期选择器，默认当天 |
| 终止日期 | 日期选择器，默认当天 |
| 查询按钮 | 点击后按选定日期范围查询数据 |
| 导出按钮 | 下拉菜单，支持导出为 JSON 或 CSV 格式 |
| 刷新按钮 | 手动刷新用量数据 |
| 总调用次数 | 选定时间范围内的 Agent 调用总数 |
| 总 Token 数 | 输入 Token + 输出 Token 汇总 |
| 预估费用 | 基于 Token 用量计算的成本预估 |
| 趋势图 | 按日维度的 Token 用量柱状图 |
| 用户分布表 | 按用户维度的用量统计（仅管理员可见全部） |

### 4. 权限控制

| 功能 | 管理员 | 普通用户 |
|------|--------|----------|
| 查看 Langfuse 状态 | ✓ | ✓ |
| 打开 Langfuse 控制台 | ✓ | ✗ |
| 配置 Langfuse 服务 | ✓ | ✗ |
| 查看所有用户用量 | ✓ | ✗ |
| 查看个人用量 | ✓ | ✓ |
| 按用户筛选 | ✓ | ✗ |
| 导出所有用户 Trace | ✓ | ✗ |
| 导出个人 Trace | ✓ | ✓ |
| 同步到 Langfuse 账号 | ✓ | ✗ |

### 4.1 用户隔离设计

**单项目 + user_id 隔离策略**：

- 所有用户的 Trace 存储在同一个 Langfuse 项目中
- 每个 Trace 携带 `user_id` 和 `session_id` 字段
- 管理员在 Langfuse 控制台可按 `user_id` 筛选定位问题
- 普通用户只能通过 SunnyAgent 界面查看自己的用量（后端 API 过滤）

### 5. 交互说明

1. **日期选择**：默认起始日期和终止日期均为当天，用户可修改后点击"查询"
2. **配置面板**：点击"配置"按钮展开/收起，仅管理员可见
3. **验证连接**：输入 URL 后点击验证，显示连接结果和延迟
4. **刷新数据**：点击刷新按钮手动触发数据同步
