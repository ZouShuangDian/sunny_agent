# SunnyAgent 部署资源规划

## 1. 框架与监控说明

### 1.1 执行引擎资源消耗

当前系统使用**自研执行引擎**（运行在 FastAPI 主服务进程内）：

| 组件 | 单请求内存 | CPU 消耗 | 说明 |
|------|-----------|----------|------|
| **IntentEngine** | 10-20MB | 低 | LLM 调用 + JSON 解析 |
| **L1 FastTrack** | 20-50MB | 低 | max 3 步，Milvus 检索 + 工具调用 |
| **L3 ReAct** | 50-200MB | 中-高 | max 20 步，循环 LLM 调用 |
| **Context 压缩** | 30-50MB | 中 | Level 2 摘要时额外 LLM 调用 |
| **工具执行** | 10-100MB | 变化 | 取决于工具类型 |

**并发资源估算**：

| 场景 | L1 请求占比 | L3 请求占比 | 平均单请求内存 | 总内存需求 |
|------|------------|------------|---------------|-----------|
| 100人并发 | 60% | 40% | ~80MB | **8GB** |
| 500人并发 | 60% | 40% | ~80MB | **40GB** |

> 执行引擎资源已计入 FastAPI 主服务规格中

**优势**：无第三方框架依赖，无额外许可费用，资源消耗可控

### 1.2 可观测性
- **Prometheus + structlog** - 已实现，无额外成本
- **Langfuse** - 私有化部署，用于 LLM Trace 追踪、Token 统计、评估

### 4.9 DeepSeek-V3 私有化部署资源估算

#### 模型规格

| 模型 | 参数量 | 激活参数 | 显存需求 (FP8) | 上下文 | 说明 |
|------|--------|----------|----------------|--------|------|
| **DeepSeek-V3** | 671B | 37B | 80-160GB | 128K | MoE 架构，L1/L3 统一使用 |

> 注：DeepSeek-V3 采用 MoE (Mixture of Experts) 架构，每次推理仅激活 37B 参数，推理效率高

#### 单次请求 Token 消耗

| 组件 | 输入 Token | 输出 Token | 调用次数 | 说明 |
|------|-----------|-----------|----------|------|
| **IntentEngine** | 500-1000 | 100-200 | 1 | 意图识别 |
| **L1 FastTrack** | 1000-2000 | 200-500 | 1-3 | 快速响应 |
| **L3 ReAct (单步)** | 2000-4000 | 500-1000 | 5-20 | 复杂推理 |
| **Context 压缩** | 3000-5000 | 500-800 | 0-2 | Level 2 摘要 |

| **FP8** | A100 80GB × 8 | 8-16 | 1200-1600 tokens/s | 高并发 |

| **FP8** | H100 80GB × 8 | 12-24 | 2000-2800 tokens/s | 旗舰配置 |
